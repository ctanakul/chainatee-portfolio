SAWYER CATCH'EM ALL

 

 

Overview

The goal of this project is to operate Sawyer, the manufacturing robot by Rethink Robotics, to catch a ball thrown at him. The hardware used are Sawyer and a stereo camera, with RGB and depth sensor. The system functions in 3 main parts, detecting the ball position, predicting the final position of the ball and the moving the arm with Jacobian based endpoint control algorithm to catch the ball. The system is run via ROS and codes are written in Python.

Part 1: The Ball Detection System

Just like human needs eyes to see objects, Sawyer needs an RGBD camera to detect the position of the ball. Here, we use Asus Xtion Pro run with openni2 package. They can process a stream of RGB image and depth image separately. The RGB image is the multi-dimensional array of type uint8 with the size of x*y*3 specifying the color of each pixel based on RGB. The depth image is the matrix of type uint8 with the size of x*y. The number of each array represents the physical distance of that pixel from camera frame.

Our strategy for detecting the ball is to get the pixel at the center of the ball in RGB image and then access the depth of it in depth image.

To get the pixel in RGB image, the color of the ball is tracked, the ball is then enclosed, and the approximated center is sent out. The image is first converted from ROS message to multi-dimensional array. Then, to get rid of the area with non-interesting color, we use cv2.inRange. At this point, the important area might not be fully extracted. Thus, the image is further converted into HSV image to keep only a portion of specified HSV range with the same function. Noise or small non-interesting area is eroded and the left important area, the ball, is dilated. The unmasked portion is then enclosed by a circle whose center pixel is outputted into projectPixelTo3dRay to get the unit vector pointing to that pixel from the camera center then it is scaled up by the depth of that pixel from depth image to get the approximated ball position. Users can adjust the RGB and HSV range and the size of eroding and dilating kernel with trackbars provided with the algorithm.

Right now you may notice that the position of the ball is relative to the position of the camera, but it was not relative to Sawyer. Exactly!!, we have to find that relative position.

To find Psb = Tsc.Pcb, we first get Tsc from lookupTransform and Pcb from the position of the ball we just got. Therefore, we can find Psb. To visualize the position of the ball, we send Marker message to RVIZ, a line strip links two detected positions, each represented by a sphere.

Part 2: the final position prediction

The projectile calculating algorithm estimates the future position of the ball by fitting the 3D parabola plot with RANSAC. The plot is being updated through throwing.

Part 3: Catching the ball with Jacobian based endpoint control algorithm

Approximately, a throw spends a second. Consequently, the arm must be moved within a second. Before every move, the hand will be moved to home position by sending the dictionary containing names and joint positions as a parameter in set_joint_velocities function of class Limb of intera_interface. The damped least square inverse kinematics is used to find joint velocity for each joint in robot arm while moving. We need to specify damping constant that is “large enough so that the solutions for joints are well-behaved near singularities, but if it is chosen too large, then the convergence rate is too slow.” This method focuses on minimizing the chance of having the set of joints that move end-effector to the unreachable point.
