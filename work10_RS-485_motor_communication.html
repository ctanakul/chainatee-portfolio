UART MOTOR COMMUNICATION

 

 

Overview

The main objective of this project is to deliver a high-level API for the communication between Linux host computer and Applied Motion's TSM23Q-4RG StepServo motor. This API will be further used to control motors for a robot in NxR (Neuroscience and Robotics Lab) at Northwestern University. Since this project was done from scratch, the steps started from hardware connection design, serial programming for POSIX operating system, to API design and writing.

Hardware connection design and testing

  Hardware connection diagram  

The motor requires the use of RS-485 standard for the communication between the host computer and the motor. The communication is done through UART protocol.

serial programming for POSIX operating system

To communicate between the host and the motor, a c code for serial programming for POSIX operating system is needed. The code is used for controlling the process of opening, closing, configuring and conducting other necessary operations such as polling or waiting until data arrives into a read buffer. Moreover, for RS-485, the flow control must be disabled since there are no wires sending hardware flow control information between the two entities. The quality of code was tested by sending a command and receiving a response from motor for 1000 times. No errors were shown.

API design and writing

  API list  

The high-level API is divided into three seperate groups of functions which are power and configurations, position control, and velocity profile control functions. Under the hood, all they do is communicating with the motor through SCL syntax by Applied Motion System.
